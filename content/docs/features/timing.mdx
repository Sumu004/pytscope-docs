The timing analyzer attributes every step to its phases — `data`, `forward`,
`backward`, `optimizer`, and (when captured) `comm` — using integer-nanosecond
timing end to end, and reports medians, p95s, and step-time variance (CV).

```bash
pytscope analyze runs/exp1
```

```text
● FINDINGS (1)
  [HIGH] Input pipeline is a bottleneck  (TIMING.DATALOADER_BOUND)
        52% of step time is spent fetching data. The accelerator is

```

## What it catches

| Rule | Fires when | Fix it points at |
|------|-----------|------------------|
| `TIMING.DATALOADER_BOUND` | `data` dominates step time | `num_workers`, `persistent_workers`, `pin_memory`, lighter transforms |
| `TIMING.BACKWARD_HEAVY` | `backward` is disproportionately slow | gradient checkpointing trade-offs, fused kernels, mixed precision |
| `TIMING.OPTIMIZER_HEAVY` | `optimizer.step` dominates | fused/foreach optimizers, larger batch to amortize |
| `TIMING.HIGH_JITTER` | step time has high coefficient of variation | host-side stalls, page faults, thermal throttling, uneven shards |
